1. Resumen ejecutivo

Este proyecto implementa una solución funcional de MLOps para automatizar el despliegue de un modelo de Machine Learning mediante contenedores Docker, GitHub Actions y servicios de Microsoft Azure. La solución permite publicar una API de inferencia en la nube y consumirla desde una página web local, demostrando el flujo completo desde el repositorio de código hasta un endpoint público operativo.

Repositorio: bairon122085-netizen / iris-mlops-deployment
Modelo: Clasificador Iris con Random Forest
API: Flask, endpoint /predict
Nube: Azure Container Registry + Azure Container Instance

2. Objetivo del sistema

El objetivo fue construir un sistema de despliegue automático que permita tomar una aplicación de inferencia de Machine Learning, empaquetarla como imagen Docker, publicarla en un registro privado de contenedores y ejecutarla en la nube mediante un endpoint accesible para usuarios finales.

La solución demuestra una arquitectura viable para que cambios futuros en la aplicación o el modelo puedan ser versionados en GitHub y procesados automáticamente por un pipeline de CI/CD.

3. Arquitectura propuesta

GitHub→ GitHub Actions→ Docker Build→ Azure Container Registry→ Azure Container Instance→ API Flask→ Página HTML

La arquitectura utiliza GitHub como repositorio central, GitHub Actions como orquestador del pipeline, Docker como mecanismo de empaquetamiento, Azure Container Registry como repositorio privado de imágenes y Azure Container Instance como servicio de ejecución del contenedor. Finalmente, una página HTML local consume el endpoint público para realizar predicciones.

5. Implementación de la API de inferencia

La API fue construida con Flask. Carga el modelo entrenado mediante joblib, recibe cuatro variables numéricas de entrada correspondientes a las dimensiones de la flor Iris y retorna la clase predicha: setosa, versicolor o virginica. 

POST /predict
Content-Type: application/json

{
  "sepal_length": 5.1,
  "sepal_width": 3.5,
  "petal_length": 1.4,
  "petal_width": 0.2
}

Además, se habilitó CORS con flask-cors para permitir que una página HTML local pudiera consumir la API desplegada en Azure.

6. Contenerización con Docker

La aplicación se empaquetó en una imagen Docker basada en python:3.11-slim. El contenedor instala las dependencias, copia los archivos de la aplicación, expone el puerto 5000 y ejecuta app.py. 

FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 5000
CMD ["python", "app.py"]
